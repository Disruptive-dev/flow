 1. QUE ES SPECTRA FLOW
================================================================================

Spectra Flow es una plataforma SaaS multi-tenant premium que consolida en una
sola app: prospeccion automatizada con IA, CRM comercial, Email Marketing
(estilo Brevo), analisis de performance y gestion multi-cliente con roles.

Lema: "Find. Clean. Activate. Convert." (Encontrar, Limpiar, Activar, Convertir)

Propuesta de valor unica:
- Encuentra prospectos B2B con IA (Google Maps + LinkedIn)
- Limpia y califica leads automaticamente via scoring con IA
- Activa leads en secuencias de email marketing personalizadas
- Convierte leads a oportunidades en un CRM completo con pipeline visual

Target: agencias de marketing, equipos comerciales B2B, consultores, PyMEs que
quieren automatizar su prospeccion y ventas desde una sola plataforma.

 2. STACK TECNOLOGICO COMPLETO
================================================================================

--- INFRAESTRUCTURA / DEPLOYMENT ---
* Hosting: DigitalOcean (Droplet propio del cliente)
* Orquestacion: EasyPanel (panel de control sobre Docker en DigitalOcean)
* Dominio principal: spectra-metrics.com
* Control de versiones: GitHub (push desde boton "Save to GitHub" de Emergent)
* CI/CD: Rebuild manual en EasyPanel post-push a GitHub
* Contenedores: Docker + docker-compose (3 servicios: frontend, backend, MongoDB)

--- FRONTEND ---
* Framework: React 19
* Build: Create React App (CRA) customizado
* Routing: react-router-dom v6
* Estado: Context API (AuthContext, LanguageContext, DemoContext)
* HTTP Client: Axios con interceptors (Bearer + refresh token + 402 handler)
* UI library: Shadcn/UI (sobre Radix UI primitives)
* Styling: Tailwind CSS + CSS variables
* Iconos: lucide-react
* Charts: Recharts (Bar, Line, Area, Pie, Funnel)
* Toasts: sonner
* Fonts: Inter + Playfair Display (via Google Fonts)
* Drag & Drop: HTML5 Drag and Drop API (CRM Kanban)
* Servidor estatico en produccion: nginx

--- BACKEND ---
* Framework: FastAPI (Python 3.11)
* Driver MongoDB: Motor (async)
* Auth: JWT (python-jose + bcrypt) con cookies HttpOnly + fallback Bearer
* Password hashing: bcrypt
* Email transaccional: Resend Python SDK
* HTTP client externo: httpx (async)
* Validacion: Pydantic v2
* Uploads: fastapi.UploadFile + StaticFiles mount en /api/uploads
* CORS: starlette CORSMiddleware
* Middleware custom: enforce_trial_expiry (bloquea tenants con trial vencido)
* Logger: logging standard de Python

 4. MODULOS Y FEATURES (DETALLE COMPLETO)
================================================================================

### 4.1 AUTENTICACION Y ONBOARDING
- Login con email/password (JWT + cookie HttpOnly + refresh token 7 dias)
- Registro de nuevo tenant: crea user + tenant, asigna plan=trial,
  trial_ends_at = +15 dias, modules = {leads:true, crm:true, rest:false}
- Forgot password: POST /api/auth/forgot-password -> manda email con link JWT
- Reset password: POST /api/auth/reset-password con token JWT + new_password
- Logout: borra cookies + dispara evento
- Refresh token automatico via interceptor axios ante 401
- Roles jerarquicos: super_admin > tenant_admin > operator
- Cambio de password desde perfil (requiere password actual)
- Actualizacion de perfil: nombre, telefono, cargo, avatar

### 4.2 MIDDLEWARE DE TRIAL (GENTE QUE NO PAGA, NO PASA)
- Middleware FastAPI que intercepta TODA la API (/api/*)
- Si el tenant tiene plan="trial" y trial_ends_at < now -> retorna 402
  con body {detail: "Tu trial de 15 dias expiro...", code: "trial_expired"}
- Excepciones: /api/auth/*, /api/tenant/status, /api/tenant/request-upgrade,
  /api/tenant/track-event, /api/admin/*, /api/uploads/*
- Super admin bypassea siempre
- Frontend: interceptor axios dispara evento "trial-blocked" -> modal con
  boton "Activar plan Pro" que envia solicitud a info@spectra-metrics.com

### 4.3 TRIAL COUNTDOWN BANNER
- Componente <TrialBanner /> en AppLayout, visible en toda la app
- Lee /api/tenant/status cada 5 min
- Color-coded: azul (>7 dias), ambar (4-7 dias), rojo (<=3 dias)
- Dismissible por sesion (sessionStorage)
- CTA "Activar Pro" -> dialog con 5 beneficios + solicitud
- Al abrir dialog dispara evento "upgrade_dialog_opened"
- Al enviar solicitud dispara "upgrade_requested" + email a admin

### 4.4 EMBUDO DE CONVERSION (SUPER ADMIN)
- Endpoint: GET /api/admin/conversion-funnel (solo super_admin)
- Metricas:
  * total_tenants (trial vs paid)
  * banner_shown_unique (tenants unicos que vieron el banner)
  * dialog_opened_unique (tenants unicos que abrieron el dialog)
  * requested_unique (tenants unicos que solicitaron upgrade)
  * dialog_open_rate, request_rate, end_to_end_rate
  * recent_requests (ultimas 20 solicitudes pendientes)
- UI: Card en /admin/tenants con 5 KPIs + lista de ultimas solicitudes
- Eventos trackeados en conversion_events:
  * trial_banner_shown (1x por sesion por tenant)
  * trial_banner_dismissed
  * upgrade_dialog_opened
  * upgrade_requested
  * trial_expired_blocked

### 4.5 SPECTRA PROSPECCION
- Buscador de prospectos: form con provincia, ciudad, categoria, cantidad
- Source selector: google_maps | linkedin
- Modo n8n: si n8n esta configurado, manda webhook y espera callback

 6. FLUJOS PRINCIPALES DE USUARIO
================================================================================

### FLUJO 1: REGISTRO NUEVO CLIENTE
1. Usuario va a /login -> click "Registrarse"
2. Completa nombre, organizacion, email, password -> submit
3. Backend crea tenant con plan=trial, trial_ends_at=+15d, modulos=leads+crm
4. Login automatico, redirige a /
5. Banner superior muestra "Te quedan 15 dias de trial"
6. Solo ve en sidebar: Dashboard, Leads, CRM, Analisis, Config
7. Cuando hace accion, se trackea evento trial_banner_shown

### FLUJO 2: PROSPECCION CON LINKEDIN (APIFY)
1. Settings > Integraciones > activa Apify + pega API key
2. Prospect Finder > source=linkedin, keyword="restaurantes", city="Tucuman"
3. Submit -> crea prospect_job con status=processing
4. Background: _run_apify_linkedin ejecuta actor google-search-scraper
5. Parsea resultados organicos con URL linkedin.com/company -> crea leads
6. Auto-rescore con IA en background
7. Usuario ve progress en tiempo real en /jobs/{id}
8. Leads aparecen en /leads con status=scored, source=linkedin

### FLUJO 3: RESET PASSWORD
1. Login > click "Olvide mi password"
2. Ingresa email -> POST /api/auth/forgot-password
3. Backend genera JWT de 24h + envia email con link
   https://app.spectra-metrics.com/reset-password?token=XXX
4. Usuario abre email, click link -> ResetPasswordPage
5. Ingresa nueva password 2 veces -> POST /api/auth/reset-password
6. Backend verifica JWT, actualiza password_hash
7. Redirect a /login, user ingresa con nueva password

### FLUJO 4: TRIAL EXPIRA (USER NO PAGO)
1. Dia 16: cron no hace nada, trial_ends_at ya paso
2. User intenta hacer cualquier request a /api/*
3. Middleware detecta plan=trial + trial_ends_at<now -> 402
4. Frontend interceptor dispara evento "trial-blocked"
5. Modal bloqueante aparece: "Trial expirado"

8. Trackea evento upgrade_requested
